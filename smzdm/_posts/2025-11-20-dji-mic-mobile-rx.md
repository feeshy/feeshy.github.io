2025年9月，大疆为 DJI Mic Mini 新推出了手机版接收器，到了双11，一收一发套装的价格已经卷到了 200 元以内。由于价格亲民，这款产品吸引了不少非 Vlog 用户的关注。大家购买后的用途可能也不再局限于 Vlog 拍摄，还会涵盖通话、K歌、录音混音等多元化场景。这里我结合购买后的实际体验，聊几个容易被常规视频评测博主忽略的细节。

## 边录边充

接收器侧面有一个多功能 USB-C 接口，既可以在录音的同时给手机充电、也可以监听麦克风收录的声音。

充电最大支持 5V/1A ，**不支持任何快充协议**，对于某些功耗较高的设备，就无法实现边录边充：例如，笔记本电脑往往无法识别已插入电源，不过笔记本电脑通常不止一个 C 口，所以问题不大；iPad 虽然能成功识别，但也有一定概率会弹出“不在充电”的提示。

💡**解决方法**：外接支持PD协议的扩展坞即可启用快充。如果你的扩展坞只有 USB-A 数据接口，没有 USB-C（PD供电口不算数据接口），那么可能需要花一两块的成本额外购买一个 USB A to C 的转接头。

## 有线耳机监听方案

> DJI Mic Mini 发射器支持通过蓝牙或者接收器与手机连接。由于麦克风蓝牙连接的[音质](https://www.bilibili.com/video/BV1EEzrYnET2)与[软件兼容性](https://dl.djicdn.com/downloads/DJI_Mic_Mini/20241120/compatibilitylist/Third-party_App_Compatibility_List-CHS.pdf)较差、耳机蓝牙连接的延迟较高，这里我们只讨论接收器与耳机均通过 USB-C 接口连接到手机/电脑的情况。

有线耳机存在**硬件监听**与**软件监听**两种监听方式，主要区别如下：

|      | 硬件监听                      | 软件监听                  |
| ---- | ------------------------- | --------------------- |
| 典型场景 | 直播、录像、课堂/会议扩声             | 通话、K歌、混音              |
| 实时监听 | 对软件无要求，甚至可以没有手机           | 需要软件支持实时监听            |
| 监听延迟 | 仅硬件延迟，效果接近实时              | 硬件+驱动+软件三重延迟          |
| 事后回放 | 仅扬声器，不支持耳机                | 耳机                    |
| 连接方式 | 手机 -> 接收器 -> 模拟C口耳机 / 转接线 | 手机 -> 分线器 -> 接收器 & 耳机 |

### 硬件监听

顾名思义，硬件监听是由 DJI Mic Mini 手机版接收器硬件提供的音频监听功能，不依赖于手机即可生效。在某些相机 App（如 iOS 系统自带相机）不支持软件监听甚至**干脆就不使用手机**的情况下，硬件监听就能派上用场。当然，专业的录像软件（如 Blackmagic Camera）通常自带软件监听，并不刚需硬件监听功能。

硬件监听的优势在于它能**兼容所有场景**，无论专业的还是普通的录制软件，无论有手机/电脑/相机热靴转接件，还是仅仅只是一个充电头/移动电源，缺点则是**耳机只能实时监听麦克风采集的声音**，若需回放已录制的声音，则只能通过手机扬声器播放。

⚠️ **特别注意**：DJI Mic Mini 手机版接收器的硬件监听功能仅支持传输**模拟信号**[^1]的耳机，大家在选购耳机时注意仔细甄别，避开内置芯片的数字耳机，以免无法正常使用硬件监听。

[^1]: 当年各大手机厂商陆续取消 3.5mm 耳机接口、改用 C 口时，曾面临传输模拟信号与数字信号之争。由于模拟音频成本更高且并非 USB-C 接口的强制标准、传输易受电磁干扰，导致兼容性不佳、底噪明显，迅速被使用更便捷的蓝牙耳机及音质更佳的数字耳机所取代。如今市面上50元以上的 USB-C 接口有线耳机多为内置 DAC 的数字款，模拟 C 口耳机已相当罕见。

DJI旗舰店的客服推荐的 C 口模拟耳机是「OPPO MH135-5」。此外，经过我的实测，普通 3.5mm 接口耳机搭配原道 4.9 元的那款转接线也能实现硬件监听。原道家转接线型号较多，注意别买错成内置芯片的款式。

以下是硬件监听的两种连接方式。

- **常规流（左）：** 直插手机/电脑，即可在软件进行音频录制的同时，提供零延迟的有线耳机硬件监听。
- **独立供电流（右）：** 如果不需要录制，纯粹用于课堂/会议现场扩声，那么大可不必将手机版接收器连接到手机上，而只需将其接到充电头、移动电源等任何能提供 USB 供电的设备上即可。许多教师在面对多媒体教室扩声需求时，往往误以为必须购买价格更高的「全能版接收器」套装。实际上，「全能版」与「手机版」在音频架构上并无二致，本质区别仅在有无内置的电池与 3.5mm 音频接口。利用右图的独立供电 + 音频转接方案，即可将手机版接收器接入教室的功放系统，虽然多了外接供电与转接的步骤，却能以更低的成本实现完全相同的无线扩声效果。

### 软件监听

相比之下，软件监听更适用于**通话**、**K 歌**、**混音**等需要将麦克风监听与系统回放叠加的场景，而且只需使用常见的 C 口数字耳机或者 3.5mm 接口耳机。

软件监听的优势在于能够直接在耳机内回放手机/电脑系统的声音，避免了外放导致的录制串音。缺点则是**依赖软件适配，延迟可能高于硬件监听**，且会受到**接口数量的物理限制**：如果录制软件本身不支持监听，你就无法监控声音是否录爆；此外，手机、平板这类移动设备通常只有一个 C 口，无法直连两个设备。此时需要配合**分线器**扩展出额外的数据 C 口或 3.5mm 耳机口。

🛑 **选购提醒**：请确保分线器具有**两个数据 C 口**，或**一个数据C口 + 一个数据A口**，或**一个数据 C 口 + 一个 3.5mm 接口**。注意别买成那种仅供充电的「一充一听」式分线器。

## 驱动兼容性

将手机版接收器插入 iOS、Android 或 macOS 设备后，可直接通过系统默认驱动运行，并被支持 USB 麦克风的软件识别。

相比之下，Windows 由于历史遗留问题，存在多种音频驱动标准。不论是较新的 WASAPI 驱动，还是专业音频的 ASIO 驱动，手机版接收器都不支持，而只兼容较旧的 MME 驱动标准。这导致其在Windows上的**软件监听延迟爆炸**，且部分舍弃了旧版驱动标准的专业音频软件无法检测到该硬件。例如，在我的 Windows 电脑上，Bitwig Studio 就无法检测到接收器，而 OBS、Cakewalk Sonar 及 Ableton Live 则能正常使用。

## 手机App

对 DJI Mic Mini 设置增益、开启低切以及升级固件，需要通过配套的 DJI Mimo App 进行。App 还内置了完整的视频剪辑工具以及各类推广信息，安装后体积超过 2GB 。从这一点也能看出，大疆对旗下麦克风的定位依然锁定在 Vlog 用户身上。

但对于像我这样只有麦克风、没有大疆相机的用户来说，这些冗余功能只是白白占据了我手机的存储空间。我觉得，如果大疆能专门推出一个不带视频剪辑的精简版「DJI Mic」App，功能可以参考舒尔的 Motiv Audio ，将软件体积控制在 200MB 左右，录音体验无疑会更好。
